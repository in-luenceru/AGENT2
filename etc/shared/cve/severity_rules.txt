# Vulnerability Severity Classification Rules
# Format: CVSS_RANGE|SEVERITY_LEVEL|ACTION|ALERT_PRIORITY

# Critical Vulnerabilities (CVSS 9.0-10.0)
9.0-10.0|CRITICAL|IMMEDIATE_ACTION|HIGH|Generate immediate alert and block if possible
8.5-8.9|HIGH|URGENT_UPDATE|HIGH|Schedule urgent maintenance window within 24h
7.0-8.4|HIGH|PRIORITY_UPDATE|MEDIUM|Schedule maintenance within 72h
4.0-6.9|MEDIUM|PLANNED_UPDATE|LOW|Include in next scheduled maintenance
0.1-3.9|LOW|MONITOR|INFO|Monitor and assess during regular review

# Special Case Patterns
BACKDOOR|CRITICAL|IMMEDIATE_QUARANTINE|HIGH|Isolate affected systems immediately
RCE|CRITICAL|IMMEDIATE_ACTION|HIGH|Remote code execution - immediate mitigation
PRIVILEGE_ESCALATION|HIGH|URGENT_UPDATE|MEDIUM|Local privilege escalation - priority update
DENIAL_OF_SERVICE|MEDIUM|PLANNED_UPDATE|LOW|Service availability impact - planned update
INFORMATION_DISCLOSURE|MEDIUM|PLANNED_UPDATE|LOW|Data exposure risk - planned update

# Package Priority Classification
KERNEL|CRITICAL|IMMEDIATE_ACTION|HIGH|Kernel vulnerabilities require immediate attention
SSH|CRITICAL|IMMEDIATE_ACTION|HIGH|SSH vulnerabilities are critical for access security
WEB_SERVER|HIGH|URGENT_UPDATE|MEDIUM|Web server vulnerabilities affect service availability
DATABASE|HIGH|URGENT_UPDATE|MEDIUM|Database vulnerabilities affect data security
CONTAINER|HIGH|URGENT_UPDATE|MEDIUM|Container vulnerabilities affect isolation
LIBRARY|MEDIUM|PLANNED_UPDATE|LOW|Library vulnerabilities require assessment

# Alert Generation Rules
CRITICAL_THRESHOLD=3|If 3+ critical vulnerabilities detected, generate summary alert
HIGH_THRESHOLD=10|If 10+ high vulnerabilities detected, generate summary alert
SCAN_FREQUENCY=daily|Daily vulnerability scans
REPORT_FREQUENCY=weekly|Weekly vulnerability reports
EMERGENCY_CONTACT=true|Enable emergency contact for critical vulnerabilities
